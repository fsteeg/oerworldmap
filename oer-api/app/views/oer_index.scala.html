Read

Overview
The API supports two parameters: q and t. It serves JSON.
q (required)
Query over all fields, supports elasticsearch query string syntax.
t (optional)
Filter results by the @@type field (requires exact full value).

Write

Overview
The API supports PUT requests with JSON bodies. Note: the write API requires authorization. Replace user and pass in the sample below with your credentials.
id
Append the ID for your data to the request path, e.g. PUT /oer/id123 for ID id123. You can GET the data the same way: /oer/id123.
body
The request body is expected to be a single JSON object (not a JSON array, literal, etc).
Sample
curl -X PUT -H "Content-Type: application/json" http://user:pass@@staging.api.lobid.org/oer/id123 -d '{"field":["some","json"]}'
